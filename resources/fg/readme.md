
# 立绘资源命名与使用规范

本文件夹存放游戏角色立绘的序列帧图片。所有资源均为带透明通道的 PNG 格式，采用 **文件夹分层 + 统一文件名** 的结构，方便程序按角色、姿态、尺寸和差分状态快速索引与动态组合。

## 目录结构总览

```
fg/                                  ← 资源根目录（可根据项目重命名）
├── hinami_01f/                      ← 角色名_姿态尺寸 文件夹
│   ├── hinami_01f_00000.png         ← 裸体底图（可选）
│   ├── hinami_01f_00001.png         ← 基础底图（着装状态）
│   ├── hinami_01f_00100.png         ← 脸红差分
│   ├── hinami_01f_00200.png         ← 表情差分
│   └── ...
├── hinami_01l/
│   ├── hinami_01l_00001.png         ← 该尺寸无裸体，直接从00001开始
│   └── ...
├── hinami_02ll/
│   └── ...
└── sakura_01m/
    └── ...
```

每个角色的 **姿态 + 尺寸组合** 单独建一个文件夹。文件夹内包含该组合下所有差分帧文件。

## 文件夹命名规则

**格式：**  
`角色名英文_姿态尺寸代码`

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| 角色名 | 全小写英文字母，建议与角色设定一致 | `hinami` `sakura` |
| 下划线 | 固定分隔符 | `_` |
| 姿态尺寸代码 | 两位姿态码 + 一位尺寸码 | `01f` `02ll` |

**示例文件夹名：**
- `hinami_01f` → hinami 正面对话框大头照
- `hinami_02l` → hinami 侧面全身照
- `sakura_01m` → sakura 正面中型立绘

## 姿态码与尺寸码详表

### 姿态码

| 代码 | 含义 |
|------|------|
| `01` | 正面图 |
| `02` | 侧面图 |

### 尺寸码

| 代码 | 类型 | 图像尺寸（宽 × 高） | 内容与位置说明 |
|------|------|---------------------|----------------|
| `f`  | 对话框大头照 | 1600 × 900 | 人物位于画面左下角，用于对话框旁边的半身像 |
| `l`  | 全身照 | 1281 × 2582 | 完整角色全身站立图 |
| `ll` | 中央大头照 | 1468 × 1458 | 与 `f` 用途相似，但大头居中，画面接近正方形 |
| `m`  | 中型立绘 | 1149 × 1920 | 介于全身与特写之间，适合中等距离展示 |

组合后常见代码：`01f`、`01l`、`01ll`、`01m`、`02f`、`02l`、`02ll`、`02m`。

## 文件命名规则

**格式：**  
`角色名英文_姿态尺寸代码_5位帧号.png`

- **前缀**：与所在文件夹名完全一致（`角色名_姿态尺寸代码`）
- **帧号**：固定 5 位数字（`00000`～`00500` 或更大），不足 5 位左侧补零

**示例：**  
`hinami_01l_00001.png` — 文件夹 `hinami_01l` 内的基础底图第 1 帧（该尺寸无裸体图，故从 00001 开始）

## 帧号分段与差分含义

同一文件夹内，不同帧号代表不同类型的图像层或状态变化。

| 帧号 | 用途 | 说明 |
|------|------|------|
| `00000` | **裸体底图** | 角色的裸体基础身体，**仅在部分尺寸码中存在**。若该姿态尺寸未提供裸体状态，则此帧缺失，文件夹直接从 `00001` 开始。 |
| `00001` ~ `00099` | **基础底图** | 着装状态下的关键姿态帧，可能包含多张（如微调姿势、口型同步）。**所有尺寸码至少拥有 `00001`**。 |
| `00100` ~ `00199` | **生理现象差分** | 脸红、流汗、青筋等生理效果层，通常叠加在底图上显示。 |
| `00200` ~ `00299` | **表情差分** | 眼睛开合、嘴型变化、呆毛动态、眉毛角度等面部表情层。 |
| `00300` 及以上 | **扩展差分** | 服装变化、饰品、特殊姿势等，按项目需求定义。 |

> **重要提示：**  
> - **裸体底图是可选的**：例如全身照（`l`）可能出于设计需求不绘制裸体版本，此时文件夹内没有 `_00000.png`，引擎应以 `_00001.png` 作为第一张底图。  
> - 帧号可跳跃，不要求连续。  
> - 生理现象和表情差分通常需要与对应的底图（00001 或 00000）同时加载并叠加渲染，而不是独立显示。  
> - 扩展帧号段时请在本文件中更新记录，防止编号冲突。

|------|------|

## 程序加载建议

在游戏引擎中，可以通过拼接字符串定位资源：

```
路径 = 根目录 + "/" + 文件夹名 + "/" + 文件名
文件名 = 文件夹名 + "_" + 帧号 + ".png"
```

### 底图选择逻辑（伪代码）

```python
# 确定基础底图帧号
if file_exists(f"fg/{character}_{pose_size}/{character}_{pose_size}_00000.png"):
    base_frame = "00000"   # 存在裸体底图，使用裸体
else:
    base_frame = "00001"   # 无裸体，使用着装底图

base_path = f"fg/{character}_{pose_size}/{character}_{pose_size}_{base_frame}.png"
base = load_image(base_path)

# 加载差分（示例）
blush_path = f"fg/{character}_{pose_size}/{character}_{pose_size}_00100.png"
blush = load_image(blush_path) if blush_condition else None

expression_path = f"fg/{character}_{pose_size}/{character}_{pose_size}_00250.png"
expression = load_image(expression_path) if expression_condition else None

# 按顺序叠加渲染：底图 → 脸红 → 表情
render(base, blush, expression)
```

> 若游戏支持裸体状态切换，可在运行时根据条件决定是否退回到 `00001` 着装底图。

## 注意事项

1. **尺寸绝对固定**：必须严格按上表中的宽高像素值制作，否则会出现缩放错位或比例失调。
2. **Alpha 通道必须保留**：人物以外区域必须为全透明，不得有杂色或白色背景。
3. **命名严格遵守**：角色名、下划线、姿态尺寸代码、帧号位数一个都不能错，程序依赖该规则解析。
4. **帧号管理**：新增差分内容时，从已规划的号段中取用，并在本文件末尾记录更新日志，防止与现有资源冲突。
5. **裸体底图的存在性**：美术在输出资源时需明确告知哪些尺寸码包含 `00000`，程序需在加载时进行文件存在性检查或通过配置文件声明。
6. **版本与备份**：修改现有立绘时，建议使用新的帧号（如 `00002`→`00003`），不要直接覆盖正在使用的底图，以免影响开发中的版本。
7. **根目录位置**：所有立绘资源的根目录（如 `fg/`）应与代码中的资源加载根路径保持一致，可根据项目配置修改。
